# Getting Started

With `np.Templating`, you will be able to create powerful templates to automate common manual tasks.

## Examples

The following example demonstrates how you create a **template** and use `np.Templating` tags to place dynamic **output**.

`np.Templating` supports a variety of template formats

- [Legacy Templates](template-anatomy#template-configuration---legacy)
- [Modern Templates (frontmatter format)](template-anatomy#template-configuration---frontmatter)

### Example 1: Legacy Templates

The template meta data is everything before the first separator (`---`) and everything after separator will be template content.

**line 1:** Template Name (required)

**line 2:** _blank line_

**line 3:** Separator, ending template meta data

:::note
This example was executed on 2021-12-24 and previous `date.yesterday()` and next `date.tomorrow()` days are determined by `np.Templating` internal date value which is current date, unless otherwise specified.
:::

import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';

```markdown
# Templating Example
---

And the remainder of the template will be display here ...
```

  </TabItem>

### Example 2: Modern Templates
When creating a new template in "@Templates" folder, the template header will be in `frontmatter` format, containing two fields for template `title` and `type`, followed by template body.

title - The title should be a unique name with the folder it is located. You can use the same title in multiple templates, provided they are in different folders.
type - The type field is used to identify what type of template this is.  There are currently 3 different types of templates
- `quick-note` will be used whenever you invoke the `np:qtn` command.
- `meeting-note` will be used whenever you wish to create a global meeting note (this is **not** attached to a NotePlan event)

:::info
For more information on using Quick Notes and the `np:qtn` command, click [here](/docs/templating-commands/quick-notes/).
:::

<Tabs>
  <TabItem value="template" label="Template" default>

```markdown
---
title: myTemplate
type: empty-note
---
Hello <$ - user.first %>
```

  </TabItem>
  <TabItem value="output" label="Output">

```text
Hello Michael
```

  </TabItem>

</Tabs>

### Example 3: Using np.Templating modules
In this example, we will use one of the many `np.Templating` module methods, [WebModule](/docs/templating-modules/web-module).

<Tabs>
  <TabItem value="template" label="Template" default>

```markdown
---
title: myTemplate
---
Today's Weather: <%- web.weather() %>
```

  </TabItem>
  <TabItem value="output" label="Output">

```text
Todays Weather: Fountain Valley, California, United States: ⛅️  +66°F
```

  </TabItem>

</Tabs>

:::info
The call to `<%- web.weather() %>` will use the format you have defined in [settings](/docs/settings). In this example, the settings value is blank, thus the default weather format will be used.

For more information about `np.Templating` weather, visit [Templating Modules - WebModule](/docs/templating-modules/web-module).
:::
